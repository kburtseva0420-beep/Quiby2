# Telegram Quiz Bot

Telegram-бот для чатов с квизом "Что было раньше?". 400 вопросов про технологии, гаджеты, игры, компании и многое другое!

Что умеет:

- запускать раунд квиза в чате;
- показывать вопрос с вариантами ответа;
- не повторять вопросы, пока не закончится весь пул;
- вести статистику игроков и чатов (за последний час);
- показывать лидерборд.

## Деплой на Railway (бесплатно)

### Шаг 1: Подготовка
1. Зарегистрируйтесь на [Railway.app](https://railway.app)
2. Создайте GitHub репозиторий с кодом бота

### Шаг 2: Создание проекта
1. В Railway нажмите "New Project" → "Deploy from GitHub repo"
2. Подключите GitHub и выберите репозиторий
3. Railway автоматически обнаружит Python проект

### Шаг 3: Настройка переменных окружения
В настройках проекта Railway добавьте:

```
BOT_TOKEN=ваш_токен_от_BotFather
```

**Важно:** `RAILWAY_PUBLIC_DOMAIN` и `PORT` устанавливаются автоматически!

### Шаг 4: Деплой
1. Railway запустит `python bot_webhook.py` (из Procfile)
2. Бот автоматически настроит webhook
3. Готово! Проверьте бота в Telegram

### Проверка работы
- Откройте URL проекта (например: `https://your-app.up.railway.app`)
- Должно появиться: "Quiz Bot is running!"
- Проверьте webhook: `https://api.telegram.org/bot<TOKEN>/getWebhookInfo`

### Бесплатный лимит Railway
- $5 кредитов в месяц (~500 часов работы)
- Достаточно для небольшого бота

## Локальный запуск (polling)

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Создать `.env` по примеру `.env.example` и указать токен бота.

3. Запустить:

```bash
python bot.py
```

## Структура проекта

- `bot.py` - основной бот (polling режим для локального запуска)
- `bot_webhook.py` - бот для Railway (webhook режим)
- `questions.json` - база из 400 вопросов
- `requirements.txt` - зависимости Python
- `Procfile` - команда запуска для Railway
- `railway.json` - конфигурация Railway
- `quiz.db` - SQLite база данных (создаётся автоматически)

## Команды

- `/start` или `/help` — помощь
- `/quiz` — новый вопрос в чате
- `/stats` — ваша статистика
- `/top` — топ-10 игроков в текущем чате
- `/chatstats` — статистика чата
- `/quizreset` — сброс истории вопросов в чате

Вопросы лежат в `questions.json`. В одном и том же чате вопрос не повторяется автоматически. Когда база исчерпана, бот просит добавить новые вопросы или выполнить `/quizreset`.

## Альтернативы Railway

- **Render.com** - бесплатный тариф
- **Fly.io** - бесплатные ресурсы  
- **Vercel** (для легких ботов)

## База данных

Бот использует SQLite (`quiz.db`). На Railway файл пересоздаётся при каждом деплое.

Для постоянного хранения рекомендуется:
- Railway PostgreSQL (платно)
- Supabase (бесплатный тариф)
- MongoDB Atlas (бесплатный тариф)
